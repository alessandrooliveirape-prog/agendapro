# AgendaPro — Plano de Implementação para Comercialização

## Contexto
O app é um MVP funcional com admin completo (14 páginas), integrações reais (Mercado Pago, Stripe, WhatsApp), mas faltam itens críticos para venda: página pública de agendamento, segurança de pagamentos, e infra de deploy.

## Prioridades (em ordem de implementação)

### Fase 1: Segurança Crítica (BLOQUEADOR)
1. **Rate limiting** — Instalar `express-rate-limit`, aplicar em rotas de auth (`/login`, `/register`, `/reset-password`) e booking público
2. **Fix bypass de pagamento** — Remover `POST /create-checkout` (mock perigoso), proteger `POST /subscriptions/activate` com verificação de pagamento real
3. **Webhook signature verification** — Stripe: validar `stripe-signature` header; Mercado Pago: validar `x-signature` header
4. **Google OAuth verification** — Validar token com Google API antes de criar conta
5. **Sanitização de entrada** — Adicionar `express-mongo-sanitize` ou equivalente para prevenir injection

### Fase 2: Página Pública de Agendamento (BLOQUEADOR)
6. **Criar página React** `BookingPage.tsx` — rota `/agendar/:slug` que consome a API pública existente
7. **Fluxo completo**: seleção de serviço → seleção de profissional → seleção de data/horário → confirmação com dados do cliente
8. **Responsivo** — precisa funcionar bem em mobile (clientes agendam pelo celular)

### Fase 3: PIX Real
9. **Gerar payload BRCode** — implementar geração de payload PIX estático no formato EMV
10. **QR Code local** — usar biblioteca `qrcode` (já instalada) em vez de serviço externo

### Fase 4: Lembretes Automáticos
11. **Cron job** — Adicionar `node-cron` para rodar a cada 15 minutos, chamar o endpoint de lembretes automaticamente

### Fase 5: Correções do Frontend
12. **Fix rota de relatórios** — Criar endpoint `GET /reports/dashboard` que o frontend chama (ou ajustar frontend para endpoints existentes)
13. **Google login** — Placeholder funcional ou remover botão

### Fase 6: Infra de Deploy
14. **Dockerfile** para backend e frontend
15. **docker-compose.prod.yml** com nginx reverse proxy
16. **Scripts de deploy** para Vercel/Railway
